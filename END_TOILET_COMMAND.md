# 结束如厕指令说明文档

## 📋 指令信息

**指令码**: `EE 02 11 00 11 00 08 17`

**C代码定义**:
```cpp
const byte cmdEndToilet[8] = {0xEE, 0x02, 0x11, 0x00, 0x11, 0x00, 0x08, 0x17};
```

**指令名称**: END TOILET（结束如厕）

---

## 🎯 功能说明

结束如厕指令用于将护理床从如厕姿态恢复到正常状态，通过三个阶段顺序执行，确保动作的安全性和可靠性。

---

## 🎬 执行流程

### **前置检查**
1. **检查状态机是否空闲** - 如果忙碌则拒绝执行
2. **初始化标志位** - 清除上次动作残留的标志

---

## 📍 **阶段1** - STATE_END_TOILET_STAGE1

### 🎯 动作目标
- **M5（顶马桶电机5）→ 位置 0**
- **M6（顶马桶电机6）→ 位置 0**
- **同步运动**：两个电机同时启动，同时监测

### 📊 执行逻辑
```
启动: M5和M6同时移动到0位置
监测:
  - M5是否到达0 (容差±10)
  - M6是否到达0 (容差±10)
  - 电流保护监测

完成条件: M5 ✓ AND M6 ✓
```

### ✅ 阶段完成
- 打印: `"=> Stage 1 completed"`
- **自动触发阶段2**

## 🔑 关键特性

### 1. **顺序触发机制**
- 阶段1完成 → **自动触发**阶段2
- 阶段2完成 → **自动触发**阶段3
- 无需发送额外指令

### 2. **M5+M6同步控制**
- 阶段1中两个电机**同时启动**
- 各自**独立位置反馈**
- **都到达**目标才进入下一阶段

### 3. **容错设计**
- 如果某电机已在目标位置 → 跳过移动，直接标记完成
- 每个电机独立过流保护
- 位置判定使用容差范围（±10单位）

### 4. **状态机安全**
- 只有IDLE状态才接受指令
- 忙碌时拒绝执行，打印提示信息
- 过流保护自动中止状态机

## 🔍 与如厕指令的对比

| 特性 | 如厕指令 (0x16) | 结束如厕指令 (0x17) |
|------|----------------|-------------------|
| **阶段1** | M4→0 | M5+M6→0 (同步) |
| **阶段2** | M3→0 | M3→12500 |
| **阶段3** | M5+M6→4500 (同步) | M4→8800 |
| **并行动作** | M1反转+M2→0 (坐起) | 无并行 |
| **前置条件** | M0必须在4200-4800范围 | 无前置条件 |
| **顺序触发** | ✅ | ✅ |
| **M5+M6同步** | ✅ (阶段3) | ✅ (阶段1) |

## ⚠️ 注意事项

1. **顺序执行**：三个阶段必须按顺序完成，不可跳跃
2. **同步精度**：M5和M6通过各自的霍尔编码器独立反馈，可能存在微小偏差
3. **目标位置**：
   - M5/M6归零：安全收回顶马桶机构
   - M3→12500：可能是床体某部分的伸出位置
   - M4→8800：可能是床体某部分的特定位置
4. **电流监测**：所有运动阶段都有过流保护
5. **EEPROM保存**：各电机位置会在静止1秒后自动保存
